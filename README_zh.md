# 🚀 AI 编码工作流

一个用于开发有效 AI 编码工作流的综合框架，包含三个阶段 - **规划**、**实现**和**验证**。

## 🧠 主要思维模型

核心理念围绕**上下文工程** - 系统化地准备和组织信息，以最大化 AI 编码助手的效能。

## 📋 阶段 1：规划

### 1. 🎨 氛围规划
使用 `/primer` 斜杠命令启动你的探索：
- **新项目**：研究在线资源、类似项目，探索架构和技术栈选项
- **现有项目**：使用**代码库分析师**子代理分析和理解当前代码库
- 重点：想法、概念和可能性的非结构化探索

### 2. 📝 创建 INITIAL.md（产品需求文档）
生成详细的产品需求文档：
- **新项目**：包含支持文档参考的高级 MVP
- **现有项目**：具有集成点的重点详细需求

### 3. ⚙️ 上下文工程组件
使用斜杠命令准备这些基本元素：

- **RAG**（检索增强生成）
- **任务管理**
- **记忆系统**
- **提示工程**

#### 🛠️ 支持工具：
- Archon
- PRP 框架
- 网络搜索
- GitHub 规范工具包

### 📊 行动计划
使用 `/create-plan` 斜杠命令基于你的 INITIAL.md 和上下文工程设置生成结构化实现策略。

## ⚡ 阶段 2：实现

### 🎯 逐任务执行
- 使用 `/execute-plan` 斜杠命令系统化地执行你的行动计划
- 遵循规划阶段创建的结构化计划
- 利用上下文工程基础

### 🔍 信任但验证
监控 AI 助手确保它：
- 正确使用 MCP 服务器
- 读取/编辑适当的文件
- 正确利用任务管理
- 产生清晰的"思考"标记显示理解

## ✅ 阶段 3：验证

### 📊 代码审查过程
**AI 助手验证**：
- 使用**验证器**子代理执行自动代码审查
- 运行单元测试
- 运行集成测试

**人工验证**：
- 战略监督
- 手动测试

## 🔧 关键组件

### 🌐 全局规则
- 子代理协调（**代码库分析师**和**验证器**）
- 斜杠命令集成（`/primer`、`/create-plan`、`/execute-plan`）
- 跨阶段的一致工作流

### 🎯 斜杠命令参考
- **`/primer`**：使用探索提示初始化氛围规划阶段
- **`/create-plan`**：从 PRD 生成结构化行动计划
- **`/execute-plan`**：系统化实现创建的计划

### 🤖 子代理
- **代码库分析师**：专门理解和分析现有代码库
- **验证器**：专注于系统化代码审查和质量保证

### 🏆 成功因素
- **结构化方法**：每个阶段都建立在前一个阶段之上
- **上下文准备**：彻底设置实现更好的 AI 性能
- **迭代优化**：在每个步骤中信任但验证
- **工具集成**：利用专门工具处理特定任务

这个框架将临时的 AI 交互转变为系统化、可重复的过程，持续产生高质量的代码和文档。🎉
